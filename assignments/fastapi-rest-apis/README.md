# 📘 Assignment: Building REST APIs with FastAPI

## 🎯 Objective

Learn how to build modern, fast REST APIs using the FastAPI framework. You will create a complete API with multiple endpoints, request/response handling, data validation, and error management using FastAPI's powerful features.

## 📝 Tasks

### 🛠️ Create a Todo API with FastAPI

#### Description
Build a RESTful API for managing a todo list application. Your API should handle creating, reading, updating, and deleting todos (CRUD operations). Implement proper request validation and error handling using FastAPI.

#### Requirements
Completed API should:

- Define a FastAPI application with multiple endpoints
- Implement GET endpoint to retrieve all todos or a specific todo by ID
- Implement POST endpoint to create a new todo with validation
- Implement PUT endpoint to update an existing todo
- Implement DELETE endpoint to remove a todo
- Use Pydantic models for request/response data validation
- Return appropriate HTTP status codes (200, 201, 404, 422, etc.)


### 🛠️ Add Advanced Features to Your API

#### Description
Enhance your todo API with more sophisticated features including filtering, pagination, and proper error handling. Add docstring documentation and explore FastAPI's automatic API documentation.

#### Requirements
Completed API should:

- Include query parameters for filtering todos (e.g., by status or priority)
- Implement pagination for retrieving todos (limit and offset)
- Add comprehensive docstrings and descriptions for endpoints
- Use FastAPI's built-in automatic documentation (Swagger UI)
- Handle edge cases with proper error responses and error models
- Follow RESTful API naming conventions and best practices
